单调栈：从暴力双循环到线性时间

进阶
栈单调栈数组
给定一个数组，如何高效地找到每个元素右边第一个比它大的元素？

直觉理解

想象你有一排身高不同的人站成一列，每个人都在找自己右边第一个比自己高的人。暴力做法是每个人往后看，复杂度O(n²)。单调栈的思路是：让这些人按身高从矮到高排成一队（栈），但只允许矮的人站在前面。当新来一个人时，如果这个人比队尾的人高，那么队尾那个人的“下一个更高的人”就是新来的人，于是队尾的人就可以离开队伍（出栈）并记录答案。新来的人继续和新的队尾比较，直到队尾的人比他高或者队伍空了，然后新来的人自己站到队尾。这样每个人最多入栈一次、出栈一次，总时间O(n)。本质上，单调栈通过维护一个单调递减的序列，把“向后看”的暴力过程变成了“向前比较”的线性扫描。

算法思路

1. 初始化一个空栈（存数组下标），以及一个结果数组ans，长度与nums相同，初始值全部设为-1（表示没找到）。 2. 从左到右遍历数组nums，对于当前下标i和值num = nums[i]： - 当栈非空 且 nums[栈顶] < num 时，说明栈顶元素找到了它的下一个更大元素，弹出栈顶下标top，设置ans[top] = num。 - 重复上述步骤直到栈空或栈顶元素的值 >= num。 - 将当前下标i压入栈。 3. 遍历结束后，栈中剩余的下标对应的元素没有下一个更大元素，ans中已经默认为-1，无需额外操作。 4. 返回ans。 关键点：栈中存的是下标，而不是值，因为我们需要通过下标更新答案数组。比较时用值比较。单调性为栈底到栈顶严格递减（即栈顶最小）。

Python 示例

def next_greater_element(nums):
    """
    返回每个元素右边第一个比它大的元素，没有则返回-1
    示例: nums = [2,1,5,3,4] -> [5,5,-1,4,-1]
    """
    n = len(nums)
    ans = [-1] * n          # 初始化答案数组，默认-1
    stack = []              # 单调递减栈，存下标

    for i in range(n):
        # 当前元素比栈顶元素大时，栈顶元素的下一个更大元素就是当前元素
        while stack and nums[stack[-1]] < nums[i]:
            top = stack.pop()
            ans[top] = nums[i]
        stack.append(i)     # 当前下标入栈

    return ans

# 测试
if __name__ == "__main__":
    test = [2, 1, 5, 3, 4]
    print(next_greater_element(test))  # 输出 [5, 5, -1, 4, -1]

    test2 = [1, 2, 3, 4, 5]
    print(next_greater_element(test2)) # 输出 [2, 3, 4, 5, -1]

    test3 = [5, 4, 3, 2, 1]
    print(next_greater_element(test3)) # 输出 [-1, -1, -1, -1, -1]

C++ 对照

#include <vector>
#include <stack>
#include <iostream>
using namespace std;

vector<int> nextGreaterElement(vector<int>& nums) {
    int n = nums.size();
    vector<int> ans(n, -1);
    stack<int> st;  // 存下标

    for (int i = 0; i < n; ++i) {
        while (!st.empty() && nums[st.top()] < nums[i]) {
            int top = st.top();
            st.pop();
            ans[top] = nums[i];
        }
        st.push(i);
    }
    return ans;
}

int main() {
    vector<int> test = {2, 1, 5, 3, 4};
    vector<int> res = nextGreaterElement(test);
    for (int x : res) cout << x << " ";  // 输出 5 5 -1 4 -1
    return 0;
}

复杂度分析

时间复杂度：O(n)，每个元素最多入栈一次、出栈一次，while循环总执行次数不超过n。空间复杂度：O(n)，栈最多存储n个下标，答案数组也占用O(n)。

易错点

  1. 栈中存的是下标而不是值：新手容易直接存值，导致无法正确更新答案数组（因为需要下标来定位）。正确做法是存下标，比较时用nums[栈顶]与当前值比较。
  2. 比较条件用 < 还是 <=：题目要求“下一个更大元素”，通常指严格大于，所以用 < 触发弹出。如果用 <=，则相等元素也会被弹出，导致相等元素的下一个更大元素被错误地赋值为当前元素（实际应该找严格大于的）。务必根据题意确认。
  3. 忘记处理栈中剩余元素：初始化答案数组为-1即可，遍历结束后栈中剩余元素自然保持-1，无需额外循环。但若题目要求返回索引差（如每日温度），则需额外处理。

延伸练习

  1. 下一个更大元素 II（循环数组）：给定一个循环数组（最后一个元素的下一个元素是第一个），求每个元素的下一个更大元素。提示：将数组复制一份接在后面，或者用取模方式遍历两倍长度。
  2. 每日温度：给定一个温度列表，返回每个位置需要等待多少天才能遇到更高的温度。本质是找右边第一个更大元素的下标差，用单调栈存下标即可。
