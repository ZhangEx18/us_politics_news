一文彻底弄懂二分查找的边界与模板

进阶
二分数组
如何正确设置二分查找的区间定义（开闭）和更新规则，以避免死循环和越界，高效地查找左右边界？

直觉理解

二分查找的本质，是利用数据已排序的特性，通过每次将搜索范围减半来快速定位目标。就像在电话簿里找人，你不会从头翻到尾，而是先翻到中间，看名字比目标大还是小，从而决定看前半本还是后半本。但“中间”有好几个：这个人可能刚好出现多次，你可能想找第一个或最后一个。这时，“边界”问题就来了。关键在于定义一个“循环不变量”：在整个查找过程中，始终维护一个保证包含答案的“范围”，并用明确的规则（比如左闭右闭或左闭右开）来描述它。每次循环，我们都通过这个规则安全地“切掉”不可能的一半，直到范围缩小到一个点。理解并固定一种区间定义，是避免“越界”和“死循环”的根本。

算法思路

解决二分查找边界问题，核心是统一区间定义并保持逻辑一致。我们通常采用两种模板：左闭右闭 `[left, right]` 和左闭右开 `[left, right)`。以下是基于“左闭右闭”模板的通用解题框架。 1. **初始化**：设定搜索区间。对于在数组中查找某个值的左边界（第一个出现的位置），`left = 0`，`right = len(nums) - 1`。注意 `right` 是有效下标。 2. **循环条件**：`while left <= right`。这是因为当 `left == right` 时，区间 `[left, right]` 仍包含一个有效元素，需要检查。 3. **计算中点**：`mid = left + (right - left) // 2`。这种方式避免了 `(left + right)` 可能的整数溢出（在C++等语言中尤为重要）。 4. **比较与更新**：这是查找边界的关键。 * **查找左边界**：若 `nums[mid] >= target`，说明左边界可能在 `mid` 或其左侧，因此令 `right = mid - 1`。这里 `right` 更新为 `mid-1`，是因为 `mid` 已经被检查过，并且我们继续向左搜索更小的索引。即使 `nums[mid] == target`，我们也不停止，因为我们要找的是第一个。 * **查找右边界**：若 `nums[mid] <= target`，说明右边界可能在 `mid` 或其右侧，因此令 `left = mid + 1`。 5. **终止与结果**：循环结束时，`left > right`。对于**查找左边界**，`left` 指向第一个大于等于 `target` 的位置。需要额外判断该位置是否越界或值不等于 `target`。对于**查找右边界**，`right` 指向最后一个小于等于 `target` 的位置。 总结：固定区间定义，用 `<=` 作为循环条件，并严格遵守“收缩区间时，将检查过的 `mid` 排除在外”的原则，是构建正确模板的基石。

Python 示例

from typing import List

def search_left_boundary(nums: List[int], target: int) -> int:
    """
    查找target在升序数组nums中第一次出现的索引（左边界）。
    如果不存在，返回它本应插入的位置（即第一个大于target的索引）。
    使用左闭右闭区间 [left, right]。
    """
    left, right = 0, len(nums) - 1
    # 循环不变量：左边界（第一个等于target的位置）一定在 [left, right] 中，或不存在（left 将指向插入点）。
    while left <= right:
        # 防止溢出（在Python中不是问题，但作为好习惯保留）
        mid = left + (right - left) // 2
        if nums[mid] >= target:
            # mid 可能是左边界，但我们要找更左的，所以收缩右边界。
            right = mid - 1
        else: # nums[mid] < target
            # mid 肯定在左边界左边，收缩左边界。
            left = mid + 1
    
    # 循环结束时，left 是第一个 >= target 的位置。
    # 检查 left 是否在数组范围内且等于 target。
    if left < len(nums) and nums[left] == target:
        return left
    else:
        # 未找到，left 即为插入点（满足题目常见要求）。
        return left

# 另一个常见变体：查找小于target的最大元素索引
def find_last_less_than(nums: List[int], target: int) -> int:
    """
    查找数组中最后一个严格小于target的元素的索引。
    如果不存在（所有元素都>=target），返回-1。
    同样使用左闭右闭区间。
    """
    left, right = 0, len(nums) - 1
    result = -1
    while left <= right:
        mid = left + (right - left) // 2
        if nums[mid] < target:
            # mid 可能是答案，记录它，并继续向右寻找更大的。
            result = mid
            left = mid + 1
        else: # nums[mid] >= target
            # 不符合要求，向左收缩。
            right = mid - 1
    return result

# 测试用例
if __name__ == "__main__":
    nums = [1, 2, 2, 2, 3, 4, 5]
    
    # 测试查找左边界
    print("--- 查找左边界 ---")
    print(f"数组: {nums}")
    print(f"目标 2 的左边界索引: {search_left_boundary(nums, 2)}") # 期望输出: 1
    print(f"目标 3 的左边界索引: {search_left_boundary(nums, 3)}") # 期望输出: 4
    print(f"目标 0 (不存在) 的左边界索引: {search_left_boundary(nums, 0)}") # 期望输出: 0 (插入点)
    print(f"目标 6 (不存在) 的左边界索引: {search_left_boundary(nums, 6)}") # 期望输出: 7 (插入点，等于数组长度)
    
    # 测试查找小于target的最大索引
    print("\n--- 查找小于目标的最大值 ---")
    print(f"数组: {nums}")
    print(f"小于 3 的最大元素索引: {find_last_less_than(nums, 3)}") # 期望输出: 3 (元素2)
    print(f"小于 1 的最大元素索引: {find_last_less_than(nums, 1)}") # 期望输出: -1 (不存在)

C++ 对照

#include <iostream>
#include <vector>
using namespace std;

// 查找左边界（第一个等于target的位置）
int searchLeftBoundary(vector<int>& nums, int target) {
    int left = 0, right = nums.size() - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2; // 防止溢出
        if (nums[mid] >= target) {
            right = mid - 1;
        } else {
            left = mid + 1;
        }
    }
    // 循环结束，left是第一个>=target的位置
    if (left < nums.size() && nums[left] == target) {
        return left;
    } else {
        return left; // 插入点
    }
}

// 查找小于target的最大元素索引
int findLastLessThan(vector<int>& nums, int target) {
    int left = 0, right = nums.size() - 1;
    int result = -1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] < target) {
            result = mid; // 记录候选答案
            left = mid + 1; // 向右找更大的
        } else {
            right = mid - 1;
        }
    }
    return result;
}

int main() {
    vector<int> nums = {1, 2, 2, 2, 3, 4, 5};
    
    cout << "--- 查找左边界 ---" << endl;
    cout << "目标 2 的左边界索引: " << searchLeftBoundary(nums, 2) << endl; // 1
    cout << "目标 3 的左边界索引: " << searchLeftBoundary(nums, 3) << endl; // 4
    
    cout << "\n--- 查找小于目标的最大值 ---" << endl;
    cout << "小于 3 的最大元素索引: " << findLastLessThan(nums, 3) << endl; // 3
    
    return 0;
}

复杂度分析

时间复杂度：O(log n)。每次迭代都将搜索区间减半，直到区间为空。对于一个长度为 n 的数组，最多需要 log₂(n) 次迭代，因此时间复杂度是对数级别的，非常高效。 空间复杂度：O(1)。算法只使用了常数级别的额外空间（几个变量 left, right, mid, result），与输入数组的大小无关。

易错点

  1. **区间定义与循环条件不匹配**：错误示例：定义区间为 `[left, right)`（右开），却使用 `while left <= right` 作为循环条件。这会导致在 `left == right` 时，区间内其实没有元素，却继续循环，可能引发越界或死循环。**修复**：如果采用左闭右开区间 `[left, right)`，循环条件应为 `while left < right`。
  2. **计算中点的方式错误**：错误示例：使用 `mid = (left + right) / 2`。在C++、Java等语言中，当 `left` 和 `right` 都很大时，`left + right` 可能导致整数溢出，产生负数，从而引发未定义行为或错误的 `mid` 值。**修复**：始终使用 `mid = left + (right - left) / 2` 来计算中点。
  3. **查找边界时，更新规则错误导致死循环**：错误示例：在查找左边界时，当 `nums[mid] == target` 时执行 `right = mid`。在 `left < right` 且 `mid == left` (例如 `left=0, right=1`) 的情况下，这会导致 `right` 始终被赋值为 `left`，`left` 无法推进，陷入死循环。**修复**：坚持“收缩区间时将检查过的 `mid` 排除在外”的原则。在左闭右闭区间下，当 `nums[mid] >= target` 时，执行 `right = mid - 1`；当 `nums[mid] <= target` 时，执行 `left = mid + 1`。

延伸练习

  1. **[基础巩固] 在排序数组中查找元素的第一个和最后一个位置**。给定一个按照升序排列的整数数组 `nums` 和一个目标值 `target`。查找给定目标值在数组中的开始位置和结束位置。如果数组中不存在目标值，返回 `[-1, -1]`。这是对左边界和右边界查找模板的直接应用。
  2. **[适度拓展] 搜索旋转排序数组**。给定一个升序排列的整数数组 `nums`，在某个未知下标处进行了旋转（例如 `[0,1,2,4,5,6,7]` 可能变为 `[4,5,6,7,0,1,2]`）。给定一个目标值 `target`，如果数组中存在这个目标值，则返回它的下标，否则返回 -1。这道题需要你在非完全有序的数组上巧妙地应用二分查找的思想，确定每次中点后，哪一部分是有序的，从而决定搜索方向。
