单调栈：从暴力到线性优化

进阶
栈单调栈数组
如何高效地找到数组中每个元素右侧第一个比它大的元素（下一个更大元素）？

直觉理解

想象你在一个排队买奶茶的队伍里，想找到你前面第一个比你高的人。你可以从队尾开始，用一个“备忘录栈”记录下从后往前看，你看到的身高是“递减”的人。当你处理到一个新人时，把他和栈顶（也就是当前记录的最高的人）比一下。如果新人比他高，说明栈顶这个人对于后面更早的人来说，已经不再是“第一个更高的”了，因为这个新人挡在了中间。于是我们把栈顶弹出（更新备忘录），继续检查新的栈顶，直到栈空或找到一个比新人高的。最后，栈顶那个人就是新人右侧第一个比他高的人，然后我们把新人自己压栈，因为现在他是队伍后面的人“可能”看到的第一个更高的人。这个“备忘录栈”始终保持着单调递减的序列，这就是单调栈的本质：通过维护一个有单调性的栈，把“回头看”的过程从 O(n) 优化到均摊 O(1)。

算法思路

我们以经典的「下一个更大元素 I」问题为例，讲解单调栈的通用解题框架： **1. 问题定义与暴力解法** 给定一个整数数组 `nums`，对于每个元素，找到它右侧第一个比它大的元素，若不存在则标记为 -1。暴力解法是双层循环：对于每个 `i`，向右扫描找到第一个 `nums[j] > nums[i]`。时间复杂度为 O(n²)，在数据量大时无法接受。 **2. 单调栈的引入** 我们优化的核心在于避免重复比较。当我们从数组的**右向左**遍历时，可以利用一个栈来“记住”那些可能成为其他元素“下一个更大元素”的候选值。为了让这个“记忆”高效，我们让栈内的元素**从栈顶到栈底严格递减**（或非递增）。 **3. 具体算法步骤** - **初始化**：创建一个空栈 `stack` 和一个结果数组 `res`（初始化为 -1）。 - **从右向左遍历**数组 `nums`： a. **维护单调性**：当栈不为空，且当前元素 `nums[i]` **大于等于** 栈顶元素 `stack[-1]` 时，说明栈顶元素不可能是 `nums[i]` 或其左边元素的“下一个更大元素”（因为 `nums[i]` 更大且在它左边）。因此，我们将栈顶元素**弹出**。 b. **记录答案**：如果此时栈不为空，那么栈顶元素就是 `nums[i]` 右侧第一个比它大的元素。将 `res[i]` 设为栈顶元素。如果栈为空，说明 `nums[i]` 右侧没有比它大的元素，`res[i]` 保持 -1。 c. **将当前元素压栈**：将 `nums[i]` 压入栈中，因为它可能成为其左侧元素的“下一个更大元素”。 - **返回结果**。 **4. 为什么是“从右向左”？** 从右向左遍历时，栈中保存的恰好是当前元素 `nums[i]` 右侧的部分元素，并且这些元素在原始顺序上是从 `i` 的右边**越来越远**的。而栈的单调递减性质保证了栈顶元素不仅是 `nums[i]` 右边第一个比它大的（在栈里），而且在原始数组中也是最近的那个。从左向右遍历也可以解决类似问题，但目标通常是找“前一个”更大元素，思路是对称的。

Python 示例

def next_greater_element(nums):
    """
    使用单调栈，从右向左遍历，找出每个元素右侧第一个更大的元素。
    :param nums: 输入的整数列表
    :return: 结果列表，对应位置为下一个更大元素，若无则为-1
    """
    n = len(nums)
    # 初始化结果数组，默认所有位置为-1
    res = [-1] * n
    # 初始化一个栈，用于存储可能成为“下一个更大元素”的候选值
    stack = []

    # 从数组末尾向开头遍历
    for i in range(n - 1, -1, -1):
        # 维护单调递减栈：如果当前元素大于等于栈顶，则栈顶元素不可能是当前或左侧元素的答案，弹出
        while stack and nums[i] >= stack[-1]:
            stack.pop()
        # 如果栈不为空，栈顶就是当前元素右侧第一个比它大的数
        if stack:
            res[i] = stack[-1]
        # 将当前元素压栈，等待被左侧元素“查询”
        stack.append(nums[i])
    return res

# 测试用例
def test():
    # 测试1: 基本情况
    print("测试1:")
    nums1 = [2, 1, 2, 4, 3]
    print(f"输入: {nums1}")
    print(f"输出: {next_greater_element(nums1)}")  # 期望: [4, 2, 4, -1, -1]
    print()
    
    # 测试2: 递减序列，所有元素右侧都没有更大的
    print("测试2:")

    print(f"输出: {next_greater_element(nums2)}")  # 期望: [-1, -1, -1, -1, -1]
    print()
    
    # 测试3: 递增序列，每个元素右侧紧邻的就是更大的

C++ 对照

#include <iostream>
#include <vector>
#include <stack>
#include <algorithm>

using namespace std;

vector<int> nextGreaterElement(vector<int>& nums) {
    int n = nums.size();
    vector<int> res(n, -1); // 初始化结果数组
    stack<int> stk; // 存储候选值的栈

    // 从右向左遍历
    for (int i = n - 1; i >= 0; --i) {
        // 维护单调递减栈
        while (!stk.empty() && nums[i] >= stk.top()) {
            stk.pop();
        }
        // 栈不为空，则栈顶为答案
        if (!stk.empty()) {
            res[i] = stk.top();
        }
        // 当前元素入栈
        stk.push(nums[i]);
    }
    return res;
}

// 测试函数
void test() {
    // 测试用例1
    vector<int> nums1 = {2, 1, 2, 4, 3};
    auto res1 = nextGreaterElement(nums1);
    cout << "测试1: ";
    for (int x : res1) cout << x << " ";
    cout << endl; // 期望: 4 2 4 -1 -1

    // 测试用例2
    vector<int> nums2 = {5, 4, 3, 2, 1};
    auto res2 = nextGreaterElement(nums2);
    cout << "测试2: ";
    for (int x : res2) cout << x << " ";
    cout << endl; // 期望: -1 -1 -1 -1 -1
}

int main() {
    test();
    return 0;
}

复杂度分析

时间复杂度：O(n)。虽然代码中有一个 `while` 循环，但每个元素最多被压入栈一次、弹出一次。所有元素的压栈和弹栈操作的总次数是 O(n)，因此均摊下来，处理每个元素的平均时间是常数级。总体时间复杂度为线性。 空间复杂度：O(n)。最坏情况下，栈需要存储所有 n 个元素（例如，输入数组严格递减时），因此空间复杂度为 O(n)。结果数组 `res` 也占用 O(n) 空间。

易错点

  1. **易错点1：搞错遍历方向或栈的单调性**。常见错误是混淆“从右向左遍历找右边更大”和“从左向右遍历找左边更大”，或者将栈的单调性写反（例如，想找递减栈却维护成了递增栈）。修复方法是明确问题目标：找右边第一个更大，就从右向左遍历并维护递减栈；找左边第一个更大，就从左向右遍历并维护递减栈。在写代码前，用小例子（如 [2,1,2,4]）手动模拟一遍栈的变化，验证正确性。
  2. **易错点2：栈中存储值还是索引**。在本题中，我们只需要知道“下一个更大元素的值”，所以栈中直接存储 `nums[i]` 的值即可。但在一些变体问题中（例如，需要知道这个更大元素的下标，或者需要计算索引差），栈中就需要存储元素的下标 `i`，而不是值 `nums[i]`。这是一个重要的设计选择。修复方法是仔细审题，根据是否需要下标信息来决定栈中存储的内容。
  3. **易错点3：边界条件与while循环条件**。`while stack and nums[i] >= stack[-1]` 这个条件中的 `>=` 至关重要。如果写成 `>`，对于重复元素，例如 `[1, 1, 1]`，答案会错误地变成 `[1, 1, -1]` 而不是 `[-1, -1, -1]`，因为相同的元素不会弹出栈，导致第一个 `1` 的答案错误地指向了第二个 `1`。修复方法是想清楚重复元素的逻辑：后面的相同元素，对于更左边的元素来说，不应该成为“更大”的候选，因此应该被弹出。
